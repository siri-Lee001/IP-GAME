# 对话式流程（6 次询问体系）

本文件把“问用户的问题模板”固化下来，供任何平台/Agent 在调用 IP-GAME 时复用。  
目标：**少来回、强确认、永远有保底路径**。

约束：
- 第一次只要“故事主题/剧本”，收到后先自动拆解能确定的信息
- 只有信息不足时才进入第二次补问（一次性 8 问）
- 必填项缺失最多追问一次：是否允许 AI 默认；同意则继续，不同意则暂停等待用户补充
- “角色图来源 / 节点图来源 / 视频来源”三段是强制确认节点（因为会决定成本/可行性/素材链路）
- 当用户已有素材时：必须允许走“使用本地素材”路径，且优先于再次生成

---

## 第一次询问：故事主题/短剧本（必问）

目的：拿到最小创作输入，让系统先自动拆解能推断的字段。

对用户输出模板：

```text
给我一句话故事主题或短剧本（<=2000字）就行。
你可以只写设定、冲突、结局想法都行；信息不够的部分我会在下一轮一次性补问。
```

系统动作（不展示给用户）：
- 自动尝试推断：故事类型、情绪、风格、互动深度、比例等
- 写入 `story.json/meta.questionnaire`（不确定的留空）
- 记录缺失字段列表（用于决定是否进入第二次询问）

---

## 第二次询问：信息不足时的一次性补问（8 问）

触发条件：第一次输入不足以确定核心字段（类型/情绪/长度/互动深度/比例/执行模式）。

对用户输出模板（可直接复制）：

```text
我已经根据你给的故事先自动拆解了一部分信息。为了继续生成影游剧本，还缺一些关键项，请你按顺序一次性回答（不想填就留空，我会用 AI 默认；必填项如果留空我只会追问一次）：

1. 视频风格（可选）：电影级CG写实 / 国风水墨 / 二次元动画 / 3D卡通 / 赛博科幻 / 手绘插画 / 其它（自填：）
2. 故事类型（必填）：仙侠 / 悬疑 / 科幻 / 现代都市 / 治愈 / 热血 / 童话奇幻 / 其它（自填：）
3. 整体情绪（必填）：轻松 / 温暖 / 紧张 / 热血 / 悲壮 / 其它（自填：）
4. 故事长度（必填）：短（3-5节点）/ 中（6-10节点）/ 长（10-15节点）
5. 互动深度（必填）：单线 / 分支剧情 / 多结局
6. 画面比例（必填）：横版16:9 / 竖版9:16
7. 执行模式（必选）：逐步确认 / 全自动化
8. 其它补充说明或注意事项（可选）：
```

追问规则（内部）：
- 必填项为空：只追问一次 `是否允许 AI 默认？（是/否）`
- 用户选“否”：暂停流程，等待用户补齐

落盘规则（内部）：
- `视频风格` → `meta.questionnaire.visual_style`
- `故事类型` → `meta.questionnaire.genre`
- `整体情绪` → `meta.questionnaire.mood`
- `故事长度` → `meta.questionnaire.length`
- `互动深度` → `meta.questionnaire.interactivity`
- `画面比例` → `meta.questionnaire.aspect_ratio`
- `执行模式` → `meta.runMode`
- `注意事项` → `meta.questionnaire.notes`

---

## 第三次询问：交付并确认“剧本与地图框架”

目的：在进入素材阶段前，让用户确认“故事结构”。

必须交付给用户确认的内容（建议按小标题输出）：
- 故事梗概（一句话 + 详细版）
- 主要角色小传（含外貌/性格/动机/关键识别点）
- 服化道关键元素
- 影游路线/地图框架（节点树/分支关系）
- 节点清单（每个节点的场景描述/对白/选项/跳转/是否结局）
- 主题色建议（可给 1-2 个备选，但此时不强制用户决定）

对用户输出结尾模板：

```text
请你确认是否要修改：
1) 角色小传
2) 分支路线/节点设置
3) 具体剧情/对白

你可以直接回复：
- “确认，进入角色图阶段”
- 或 “修改：……”（按点列出即可）
```

---

## 第四次询问：角色图来源（二选一，强制）

触发：用户确认剧本后，进入“角色定妆”阶段前。

对用户输出模板：

```text
下一步是生成/整理主要角色图（强制步骤，用来保证角色一致性）。
请选择角色图来源：
1) 直接生成（由平台/模型生成角色定妆多视角图）
2) 使用本地已有角色图（你已经有角色三视图/设定图/参考图，我将按路径映射并以此为身份锚点）
```

硬规则：
- 若用户已经明确“我有角色图”，必须优先走选项 2，不得擅自重生覆盖

若用户选 2，补问一次（必要信息）：

```text
请把角色图放到项目目录的 `assets/images/characters/`，并告诉我对应文件名（或直接发路径）。
建议每个角色至少提供：sheet（多视角合成图）或 front（正面图）；有 side/back/face 更好。
```

---

## 第五次询问：节点场景图/故事板图来源（二选一，强制）

触发：角色图确认后，进入节点图生成前。

对用户输出模板：

```text
下一步开始为每个剧情节点生成两类图片：
- 场景图 scene still
- 九宫格分镜图 storyboard still

请选择生图方式：
1) 直接生成
2) 使用本地已有图片（你提供各节点对应图片，我负责路径映射与自检）
```

硬规则：
- 两类图必须同一画面比例（16:9 或 9:16），不可混用

---

## 第六次询问：视频来源（三选一，强制）

触发：节点图确认后，进入视频阶段前。

对用户输出模板：

```text
下一步生成节点视频，请选择视频来源：
1) 视频 API 生成（你提供接口/密钥；密钥不能进仓库）
2) 静态图合成视频（本地 ffmpeg：用场景图+分镜图合成 5-15 秒 MP4，稳定保底）
3) 使用本地视频素材（你提供 mp4，我负责路径映射与自检）
```

策略建议（可选提示给用户）：
- 测试跑通优先选 2
- 质量优先且有 API 才选 1

---

## 网页主题色确认（最终一次轻询问）

触发：生成 `game.html` 前。

对用户输出模板：

```text
最后一步：网页主题色你希望用什么？
你可以给一个颜色名或色值（例如：墨绿 / 深紫 / #6E56CF）。
如果你不回答，我会根据故事类型与情绪自动决定并直接打包生成网页。
```

默认策略（内部）：
- 未回答 → AI 自动推断
- 回答了 → 写入 `ui.json` 并生成 `game.html`
